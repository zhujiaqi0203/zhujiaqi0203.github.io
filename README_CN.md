# GitHub 文件传输系统

一个完全在 GitHub Pages 上运行的简单文件传输系统，允许用户上传、分享和下载文件，无需后端服务器。

## 功能特点

- **文件上传**：拖放文件或点击选择文件进行上传。
- **文件分享**：生成唯一的下载链接，支持可选的密码保护和过期日期设置。
- **文件下载**：使用生成的链接访问和下载文件。
- **管理员访问**：输入管理员密码可查看和下载所有上传的文件。
- **GitHub 集成**：通过 GitHub API 将 GitHub 仓库作为存储后端。

## 设置说明

### 前提条件

- GitHub 账号
- 用于存储文件的 GitHub 仓库
- 具有 `repo` 权限的 GitHub 个人访问令牌

### 安装步骤

1. ** Fork 此仓库**或从此模板创建新仓库。

2. **为您的仓库启用 GitHub Pages**：
   - 进入仓库设置
   - 向下滚动到 "GitHub Pages" 部分
   - 选择您要使用的分支（通常是 `main` 或 `master`）
   - 点击 "保存"

3. **配置应用程序**：
   - 打开 `index.html` 文件
   - 更新脚本顶部的 `CONFIG` 对象，填入您的 GitHub 用户名、仓库名称和个人访问令牌：
     ```javascript
     const CONFIG = {
         GITHUB_OWNER: 'your-github-username', // 替换为您的 GitHub 用户名
         GITHUB_REPO: 'your-repo-name', // 替换为您的仓库名称
         GITHUB_TOKEN: 'your-personal-access-token', // 替换为您的个人访问令牌
         ADMIN_PASSWORD: '20130203', // 默认管理员密码
         FILES_DIR: 'files',
         SHARES_DIR: 'shares',
         MAX_FILE_SIZE: 10 * 1024 * 1024, // 10MB
     };
     ```

4. **在您的仓库中创建目录**：
   - 在仓库中创建两个目录：`files` 和 `shares`
   - 这些目录将用于存储上传的文件和分享元数据

5. **将您的更改推送到 GitHub**。

6. **访问您的应用程序**：`https://yourusername.github.io/your-repo-name`

## 使用方法

### 上传文件

1. 进入 "上传" 标签页
2. 将文件拖放到上传区域或点击选择文件
3. （可选）为文件设置密码
4. （可选）为文件设置过期日期
5. 点击 "上传文件" 按钮
6. 上传完成后，复制生成的链接并分享给他人

### 下载文件

1. 进入 "下载" 标签页
2. 将下载链接粘贴到输入框中
3. 点击 "获取" 按钮
4. 如果文件受密码保护，请输入密码
5. 点击 "下载文件" 按钮

### 管理员访问

1. 点击右上角的 "管理员" 按钮
2. 输入管理员密码（默认：`20130203`）
3. 您现在将看到所有上传文件的列表
4. 点击 "下载" 下载单个文件或 "下载全部" 下载所有文件

## 安全考虑

- **密码保护**：文件密码使用 Base64 编码，这对于敏感数据不安全。对于生产环境，请考虑实现适当的加密。
- **GitHub 令牌**：GitHub 令牌暴露在客户端代码中。对于生产环境，请考虑使用代理服务器处理 API 请求。
- **文件大小限制**：默认文件大小限制为 10MB。这可以在 `CONFIG` 对象中调整，但 GitHub 有其自己的文件大小限制。

## 限制

- **文件大小**：GitHub 对单个文件有 100MB 的大小限制。
- **速率限制**：GitHub API 有速率限制。对于大量使用，请考虑实现速率限制处理。
- **存储**：所有文件都存储在您的 GitHub 仓库中，因此您受限于仓库的存储容量。
